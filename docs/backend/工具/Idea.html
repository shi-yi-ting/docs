idea的项目结构 ​

  1. project（项目）
  2. module（模块，在模块里面新建包）
  3. package（包，也就是文件夹）（注意：包名以 com.xxx.xxx 的形式命名）
  4. class（类）

快速 get/set，toString() ​

  1. 快速get/set：在空白处右键---选择Generate---Getter and Setter，或者使用快捷键Alt+Insert；在弹出的对话框中,选择需要生成get和set方法的属性,点击OK

  2. 快速toString()：在空白处右键---选择Generate --- toString()；在弹出的对话框中,选择需要toString()的属性,点击OK

配置maven ​

对本工程进行配置：

file > settings > Build,Execution,Deployment > Build Tools > Maven（点击它） > Maven home path（本地Maven安装目录）

file > settings > Build,Execution,Deployment > Build Tools > Maven（点击它） > User Settings file（配置文件，选中之后，Local repository的路径会自动更新，如果没有自动更新就把后面的沟勾选）

Local repository：构建Maven项目之后下载的java包将会放在这里目录当中，可做修改

file > settings > Build,Execution,Deployment > Build Tools > Maven（展开它）> Importing > Automatically download（勾选之后在导入依赖包时会自动下载源码和文档，默认不勾选）

file > settings > Build,Execution,Deployment > Build Tools > Maven（展开它）> Importing > VM options for importer（-Xmx768m, 打开项目太慢时可对齐进行调整）

file > settings > Build,Execution,Deployment > Build Tools > Maven（展开它）> Runner > Delegate IDE build/run actions to Maven Run in background 两个勾选上

创建项目 ​

点击New Project > Empty Project > Next > 给项目取一个名字 > finish > 会提示在当前目录上没有你创建的项目文件夹是否需要自动创建（选择Yes）> 会有一个弹框提示不用管（close）> file > Project Structure（表示项目重构的意思） > Modules > 点击 + 号 > New Module/Import Module > Java > Next > 输入模块的名字 > finish > Yes > Ok > 等待右下角二点进度条加载满 > 在左侧点开模块文件夹 > 右键src文件夹 > New > package > 给文件夹起名字（一般为公司公司域名的反写 + 包的作用如：com.itheima.demo1）(右键选择包选择open in 选择 Explorer可以在本地打开文件夹，com.itheima.demo1是一个多级包，多级包之间是用 点 隔开的) > 右键点击包 > New > Java Class > 输入类名

创建java web工程 ​

  • file > New > Project > java Enterprise（企业项目，基于java EE）
  • Name：项目名称
  • Location：项目保存在本地的路径
  • Project template：创建工程使用的项目模板
  • Application server（应用程序服务器）：配置Tomcat, 如果这个地方显示 No application serve 可以点击后面的New > Tomcat Server创建一个 > 会有一个弹框 Tomcat Home（Tomcat的所在目录）
  • Build system（构建系统）：选择Maven
  • group：阻止名称（如：com.sxt）
  • Project SDK
  • （选完之后 Next > Version 选择Java EE版本）> dependencies（依赖，默认会选中servlet其他的不用管）
  • 在右边可以看到一个Maven
  • 启动工程直接点击右上角的启动三角即可，将进如到主页面，也就是src > main > webapp > index.jsp

web工程的工程结构 ​

  • src > main > jave：后端代码
  • resources：存放静态资源，js css 或者图片
  • test：测试代码存放的地方
  • pom.xml：记录我们用到的依赖 jar 包
  • target：项目编译后的存放目录，项目打包之后也会存放在这个目录（在右边的Maven展开下有个Lifecycle将其展开之后有个package,）

如何修改项目的访问路径 ​

关闭正在运行的项目 > 将在项目启动按钮的的左边的Tomcat展开 > 选择 Edit Configurations将会弹出Tomcat的配置弹框 > Deployment向下滑动 > Application context（项目应用的上下文）> 修改成我们想要的项目名称 > ok

如何实现热更新 ​

Edit Configurations > On frame deactivation > 选择 Update classes and resources > apply/ok

配置默认打开的浏览器 ​

Edit Configurations > After launch

Edit Configurations > Application server > 更换 Tomcat 的版本

如何引入已有的 javaWeb 项目 ​

open > 找到项目所在位置 > ok > 此时会在 Tomcat 的地方（也就是启动按钮处）会显示红叉 > Edit Configurations > Application server > 更换 Tomcat 的版本（同时 Name项也同步改一下和Tomcat的版本同步）> 改一下热更新 > 改一下项目访问路径

如何创建普通的 Maven 工程 ​

New Project > Maven > 选择好sdk > Create from archetype（原型骨架，勾选上） > org.apache.maven.archetypes:maven-archetype-quickstart > Next > 取项目名称 > 展开Artifact Coordinates > 修改GroupId（com.sxt，这个名称由自己取）> Next > 根据情况修改maven地址，如果电脑上没有自己添加的maven那么就不需要修改 > finish

在idea中如何删除已经下载的maven的依赖并重新下载 ​

  1. 打开 IntelliJ IDEA。
  2. 打开你的 Maven 项目。
  3. 在 IDEA 的右侧边栏（Project、Packages、Maven 窗口之一）中，找到并展开 "Maven" 选项。
  4. 找到你的项目下的 "Lifecycle" 选项。通常，你可以在 "Plugins" 下找到它。
  5. 在 "Lifecycle" 下，你会看到一系列 Maven 生命周期操作，包括 clean 和 install。
  6. 右键单击 clean，然后选择 "Reimport"。这将删除已下载的依赖和清理项目。
  7. 右键单击 install，然后选择 "Reimport"。这将重新下载并安装依赖。
